title: Synchro
description: Production-grade offline sync for serious native apps, built around an extension-based server architecture.
---

Synchro gives teams building serious native apps production-grade offline sync without a separate sync tier.

Its core differentiator is an extension-based server architecture: server-side sync logic runs inside PostgreSQL instead of being rebuilt in a standalone sync service or spread across application handlers.

Swift and Kotlin are first-class client engines. React Native is a bridge over them.

## Why teams choose it

- No separate sync tier. The server-side sync contract lives in the PostgreSQL extension, not in a standalone sync service.
- Native-first client engines. Swift and Kotlin own local SQLite, change capture, queueing, retry, and safe apply. React Native stays a bridge.
- Lower adoption friction. Synchro is designed for minimal required schema changes and SQL-based configuration in the database repo.
- Deterministic selective sync. Server-defined scopes, materialized membership, and explicit rebuilds make behavior predictable.
- Portable seeds. Generate client-compatible SQLite seeds from PostgreSQL for faster first start and bundled shared data.

## Current support

- PostgreSQL 18
- Swift on iOS 16+ and macOS 13+
- Kotlin on Android API 24+
- React Native bridge over the native SDKs
