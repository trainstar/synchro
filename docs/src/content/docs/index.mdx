title: Synchro
description: Offline-first sync for native apps using local SQLite, a PostgreSQL extension, and a thin HTTP adapter.
---

Synchro is a native-first sync system built around four layers:

- a shared Rust core for deterministic sync semantics
- a PostgreSQL extension that executes sync logic near the data
- a thin Go HTTP adapter
- native Swift and Kotlin SDKs, with React Native as a bridge over them

The product center is:

- local SQLite on device
- deterministic server-defined scopes
- full synced CRUD for synced tables
- thin adapters instead of server-side sync logic in application code

## Why teams use it

- local SQLite is the real product surface
- selective sync is server-defined and deterministic
- sync logic stays close to PostgreSQL instead of being reimplemented in app routers
- portable seed generation gives apps a warm-start path for schema and shared portable data

## Current support

- PostgreSQL 18
- Swift on iOS 16+ and macOS 13+
- Kotlin on Android API 24+
- React Native bridge over the native SDKs
